### rabbitmq命令行使用
	- | 步骤 | 命令                                                                      | 说明             |
	  | :- | :---------------------------------------------------------------------- | :------------- |
	  | 1  | `rabbitmqctl add_vhost <vhost_name>`                                    | 创建虚拟主机         |
	  | 2  | `rabbitmqctl add_user <username> <password>`                            | 创建新用户          |
	  | 3  | `rabbitmqctl set_permissions -p <vhost_name> <username> ".*" ".*" ".*"` | 授予用户对虚拟主机的全部权限 |
	  | 4  | `rabbitmqctl set_user_tags <username> administrator`                    | 设置用户标签为管理员     |
	  | 5  | `rabbitmqctl list_users`                                                | 查看用户列表         |
	- 在配置的 `15672` 端口用步骤4设置的管理员登录使用
	- 步骤三的3个 `"."` 分别对应：
		- **配置权限（Configure）**：允许创建/删除队列、交换机等
		- **写权限（Write）**：允许发布消息
		- **读权限（Read）**：允许消费消息
- ### scp中转使用
	- ```
	  # -3 模式下，数据会经过本地主机内存中转，不写入本地磁盘
	  scp -3 -r <source_user>@<source_host>:<source_path> <dest_user>@<dest_host>:<dest_path>
	  ```
- ### Dockerfile中ENV和ARG区别
	- | 特性              | `ARG`                      | `ENV`                      |
	  | --------------- | -------------------------- | -------------------------- |
	  | **作用域**         | 仅构建时（Build-time）           | 构建时 + 运行时（Build + Runtime） |
	  | **最终镜像**        | ❌ 不会保留在镜像中                 | ✅ 会保留在镜像的环境变量中             |
	  | `docker run` 可用 | ❌ 否                        | ✅ 是（容器内可用）                 |
	  | 默认值             | 可以设置                       | 可以设置                       |
	  | 外部传入            | `docker build --build-arg` | `docker run -e`（覆盖）        |
	- #### ARG（构建参数）
		- ```Dockerfile
		  # 定义构建参数，可设默认值
		  ARG NODE_VERSION=18
		  ARG APP_ENV=production
		  
		  FROM node:${NODE_VERSION}-alpine
		  
		  # ARG 在 FROM 之后需要重新声明才能使用
		  ARG APP_ENV
		  
		  # 构建时可用（如安装不同依赖）
		  RUN if [ "$APP_ENV" = "development" ]; then npm install; else npm ci --only=production; fi
		  ```
		- ```bash
		  docker build --build-arg NODE_VERSION=20 --build-arg APP_ENV=development -t myapp .
		  ```
	- #### ENV（环境变量）
		- ```Dockerfile
		  ENV NODE_ENV=production
		  ENV PORT=8080
		  ENV APP_HOME=/app
		  
		  # 可以组合使用
		  ENV PATH="${APP_HOME}/node_modules/.bin:${PATH}"
		  ```
		- ```bash
		  # 容器内可以读取这些变量
		  docker run myapp env | grep NODE_ENV   # 输出 production
		  
		  # 运行时可以覆盖（不影响镜像本身）
		  docker run -e NODE_ENV=staging -e PORT=3000 myapp
		  ```
	- #### 常用组合
		- ```Dockerfile
		  # 阶段1：用 ARG 接收构建参数
		  ARG APP_VERSION=1.0.0
		  ARG BUILD_DATE
		  
		  # 阶段2：用 ENV 持久化到镜像（供应用读取）
		  ENV APP_VERSION=${APP_VERSION}
		  ENV BUILD_DATE=${BUILD_DATE}
		  
		  # 应用启动时可以读取这些变量
		  CMD ["node", "server.js"]
		  ```
